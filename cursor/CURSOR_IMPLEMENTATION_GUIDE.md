# Cursor Implementation Guide

Cursor arbeitet in diesem Repository dokumentations- und spezifikationsorientiert.

## Vorgehen

1. Relevante Dokumente und ADRs lesen.
2. Betroffene Playbooks oder Schemas identifizieren.
3. Kleine, nachvollziehbare Aenderungen vornehmen.
4. Beispiele aktualisieren, wenn Formate geaendert werden.
5. Keine Produktivlogik in dieses Repository einfuegen.

## Regeln

- Windows-Kompatibilitaet beachten.
- Keine Secrets in Dateien schreiben.
- Keine Auto-Publish-Mechanismen erzeugen.
- Neue Architekturentscheidungen als ADR ergaenzen.
